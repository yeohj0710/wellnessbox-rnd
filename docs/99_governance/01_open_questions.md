# 열린 질문

## 핵심 질문

1. `master_context.md` 의 KPI 문장을 코드 metric 으로 어디까지 직접 치환할 것인가
2. recommend schema 에 evidence / citation / limitation 을 어디까지 구조화할 것인가
3. frozen eval seed 를 어떤 synthetic 분포로 확장할 것인가
4. safety rule set 의 우선 확장 범위를 무엇으로 둘 것인가
5. efficacy scoring 을 어떤 간단한 deterministic feature 로 고도화할 것인가
6. ranking 단계에서 diversity / cost / safety penalty 를 어떻게 줄 것인가
7. follow-up 질문 생성 규칙을 어떻게 표준화할 것인가
8. synthetic data generation pipeline 을 어떤 입력 grammar 로 만들 것인가
9. optional LLM layer 를 붙이기 전 필요한 최소 pass condition 은 무엇인가
10. generated OpenAPI snapshot 을 현재 유지할지, 나중에 별도 contracts 문서로 옮길지

## 현재 범위 밖이라 보류하는 질문

- 특정 제품 repo 와의 API adapter 형태
- feature flag / rollout 방식
- 제품 route 와의 연결점
- preview route 유지 여부

## 다음 세션 첫 질문

- KPI / 요구사항 재정리를 어떤 표준 형식으로 남길지부터 결정한다.
